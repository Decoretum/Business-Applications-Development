Web Inventory Order Management Software

Purpose
This software was created to create and manage product transactions for China Global Steel Trading. It could confirm orders and could store and present data in a manner that could be used for the letter of credit and manufacturing order. This software is capable of managing orders, product stock, order total costs, and the products within those orders.


Application Components



Product Management
This software allows Product management through adding products, editing products, and the deletion of products. Products are categorized through a Product object entity which includes product information and stock. Once the stock reaches 0 or less than 0, then it will not be available for being included in orders. If there are new stock available for the depleted product, then the clerk could simply edit Product information and change the stock.


Order Management
Orders are object entities that contain information about the products that were ordered within it. Orders are used to identify the products that a Consignee have ordered within an order. However, Orders aren't as straightforward as it seems in the software.

FinalOrder
An order could contain several products that a Consignee could order. In this software, Orders are known as a FinalOrder They contain fields such as place and date of issue, consignee information, time of issue, as well as other manufacturing order fields such as local vessel, shipper name, notify party, and more.

FinalOrder objects contain identification fields as well. These FinalOrder objects contain OrderedProduct objects.

Consignee
A Consignee object represents the customer or client that will order products. It contains the fields Bill of Lading identification ID and the name of the client.

OrderedProduct
For good Database practice, an object entity must be created between a "Product" object and a "FinalOrder" object in order to represent a product that was ordered within an order. This is the definition of an OrderedProduct object. It contains fields such as quantity, totalcost, remarks, and identification fields.

Notify Party
Every shipment containing multiple orders would require a Notify Party. The Notify Party would be notified when the shipment arrives at the destination. During the process of confirming an order, there would be a list of choices for available notify parties. The NotifyParty object contains information regarding a Notify Party.

In order to be more efficient, Notify parties will be added in the database of the application in order to save the information regarding these Notify Parties for future use. Hence, NotifyParty objects could be added and edited in the application.


Technical Specifications
In order to develop this software, Django (Python), SQL, and Javascript were used.

Django was used for the back-end architecture as well as the front-end views components of the application.

SQL was used to connect the SQLITE3 database of Django to a MySQL server.

Javascript was used to provide User Experience and User Interface functionalities for the application, as well as data retrievals from the fields of forms.
